## Running the Project

```bash
uv run uvicorn server:app --port 8001
```

Open `http://localhost:8001` in Chrome/Edge (requires WebGPU). No build step — vanilla ES modules served directly. `NoCacheMiddleware` in `server.py` disables browser caching for `.js/.css/.html/.wgsl` files during development.

## Architecture

Real-time Gray-Scott reaction-diffusion solver running on the client GPU via a single WebGPU compute shader. FTCS scheme (Forward-Time Centered-Space) with periodic boundary conditions.

**Module graph:**
```
main.js          → WebGPU init, RAF loop: solver.step() → renderer.draw()
├── solver.js    → 4 GPU buffers (u, v, uNew, vNew) + 1 uniform, explicit bind group layout, ping-pong dispatch
├── renderer.js  → async staging buffer readback of V field, 256×1 PNG colormap LUT, canvas putImageData
├── interaction.js → mouse brush painting (circular, writes u=0.5 v=0.5 to all 4 buffers)
├── ui.js        → DOM bindings: presets, F/k sliders, playback, resolution, keyboard (P/M/R/C), guide modal
└── presets.js   → 12 (F,k) presets + 3 seed pattern generators, loadPreset() and clearField()
```

**Compute shader** (`static/shaders/gray-scott.wgsl`): single entry point, 8×8 workgroup, 5-point Laplacian with periodic wrapping via modular arithmetic, forward Euler update. ~45 lines.

## Key Conventions

**Column-major indexing everywhere:** `idx = i * numY + j` (i=column, j=row). Used consistently across all JS modules and the WGSL shader.

**Explicit bind group layouts** (not `layout: 'auto'`): defined once in `solver.js`, two bind groups (A/B) with swapped read/write roles for ping-pong.

**Ping-pong buffer pattern:** `_flip` boolean alternates bind groups each substep. No buffer copies — just swap which group is active. Single command encoder wraps all substeps; one `queue.submit()` per frame.

**Write to BOTH buffer pairs from CPU:** When seeding from JavaScript (brush painting, preset load, clear), always write to u, v, uNew, AND vNew via `device.queue.writeBuffer()`. The solver may read from either pair depending on flip state.

**Async readback with pending flag:** `renderer.readbackPending` prevents stacking GPU→CPU readback requests. One-frame latency is expected.

**Canvas Y-flip:** Renderer maps GPU grid (j=0 at bottom) to canvas (y=0 at top): `pixelIdx = ((numY - 1 - j) * numX + i) * 4`.

## Uniform Buffer Layout (32 bytes, std140)

```
offset  0: numX (u32)    offset 16: F  (f32)
offset  4: numY (u32)    offset 20: k  (f32)
offset  8: Du   (f32)    offset 24: dt (f32)
offset 12: Dv   (f32)    offset 28: pad (u32)
```

Fixed values: Du=0.2097, Dv=0.105, dt=1.0. Stability constraint: dt ≤ 1/(4×Du) ≈ 1.19.

## Colormap

5-stop gradient (from pmneila reference): black(0) → green(0.2) → yellow(0.21) → red(0.4) → white(0.6+). Generated by `scripts/gen_colormap.py` → `static/colormaps/viridis.png`. The tight green→yellow transition at V≈0.2 creates vivid pattern edges.
