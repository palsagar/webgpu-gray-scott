## Architecture

Real-time Gray-Scott reaction-diffusion solver running on the client GPU via a single WebGPU compute shader. FTCS scheme (Forward-Time Centered-Space) with periodic boundary conditions.

**Module graph:**
```
main.js          ‚Üí WebGPU init, RAF loop: solver.step() ‚Üí renderer.draw()
‚îú‚îÄ‚îÄ solver.js    ‚Üí 4 GPU buffers (u, v, uNew, vNew) + 1 uniform, explicit bind group layout, ping-pong dispatch
‚îú‚îÄ‚îÄ renderer.js  ‚Üí async staging buffer readback of V field, 256√ó1 PNG colormap LUT, canvas putImageData
‚îú‚îÄ‚îÄ interaction.js ‚Üí mouse brush painting (circular, writes u=0.5 v=0.5 to all 4 buffers)
‚îú‚îÄ‚îÄ ui.js        ‚Üí DOM bindings: presets, F/k sliders, playback, resolution, keyboard (P/M/R/C), guide modal
‚îî‚îÄ‚îÄ presets.js   ‚Üí 12 (F,k) presets + 3 seed pattern generators, loadPreset() and clearField()
```

**Compute shader** (`static/shaders/gray-scott.wgsl`): single entry point, 8√ó8 workgroup, 5-point Laplacian with periodic wrapping via modular arithmetic, forward Euler update. ~45 lines.

## Key Conventions

**Column-major indexing everywhere:** `idx = i * numY + j` (i=column, j=row). Used consistently across all JS modules and the WGSL shader.

**Explicit bind group layouts** (not `layout: 'auto'`): defined once in `solver.js`, two bind groups (A/B) with swapped read/write roles for ping-pong.

**Ping-pong buffer pattern:** `_flip` boolean alternates bind groups each substep. No buffer copies ‚Äî just swap which group is active. Single command encoder wraps all substeps; one `queue.submit()` per frame.

**Write to BOTH buffer pairs from CPU:** When seeding from JavaScript (brush painting, preset load, clear), always write to u, v, uNew, AND vNew via `device.queue.writeBuffer()`. The solver may read from either pair depending on flip state.

**Async readback with pending flag:** `renderer.readbackPending` prevents stacking GPU‚ÜíCPU readback requests. One-frame latency is expected.

**Canvas Y-flip:** Renderer maps GPU grid (j=0 at bottom) to canvas (y=0 at top): `pixelIdx = ((numY - 1 - j) * numX + i) * 4`.

## Uniform Buffer Layout (32 bytes, std140)

```
offset  0: numX (u32)    offset 16: F  (f32)
offset  4: numY (u32)    offset 20: k  (f32)
offset  8: Du   (f32)    offset 24: dt (f32)
offset 12: Dv   (f32)    offset 28: pad (u32)
```

Fixed values: Du=0.2097, Dv=0.105, dt=1.0. Stability constraint: dt ‚â§ 1/(4√óDu) ‚âà 1.19.

## Colormap

5-stop gradient (from pmneila reference): black(0) ‚Üí green(0.2) ‚Üí yellow(0.21) ‚Üí red(0.4) ‚Üí white(0.6+). Generated by `scripts/gen_colormap.py` ‚Üí `static/colormaps/viridis.png`. The tight green‚Üíyellow transition at V‚âà0.2 creates vivid pattern edges.
